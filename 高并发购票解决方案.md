# 大麦网高并发购票解决方案

## 一、当前架构问题

```
用户请求 → Tomcat线程池 → Controller → Service → MySQL
                      ↑
                并发瓶颈：MySQL单点写入
```

**瓶颈分析：**
1. 数据库连接池耗尽
2. 乐观锁冲突导致大量重试
3. 超卖问题（最后一张票）
4. 恶意刷单

## 二、完整解决方案

### 方案1：Redis + Lua 原子操作（推荐）

```java
@Service
public class RedisTicketService {
    
    // Lua脚本：原子性扣减库存
    private static final String DECREASE_STOCK_SCRIPT = 
        "if redis.call('exists', KEYS[1]) == 1 then " +
        "  local stock = tonumber(redis.call('get', KEYS[1])) " +
        "  if stock >= tonumber(ARGV[1]) then " +
        "    return redis.call('decrby', KEYS[1], ARGV[1]) " +
        "  else return -1 end " +  // 库存不足
        "else return -2 end";      // 座位不存在
    
    public boolean lockSeat(Long showId, Long seatId) {
        String key = "seat:" + showId + ":" + seatId;
        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(DECREASE_STOCK_SCRIPT, Long.class),
            Collections.singletonList(key), "1"
        );
        return result != null && result >= 0;
    }
}
```

### 方案2：分布式锁（Redisson）

```java
@Service
public class DistributedLockService {
    
    public boolean tryLockSeat(Long showId, Long seatId) {
        String lockKey = "seat:lock:" + showId + ":" + seatId;
        RLock lock = redissonClient.getLock(lockKey);
        
        try {
            // 尝试获取锁，最多等待3秒，锁定后10秒自动释放
            boolean acquired = lock.tryLock(3, 10, TimeUnit.SECONDS);
            if (acquired) {
                return lockSeatInDB(showId, seatId);
            }
            return false;
        } catch (InterruptedException e) {
            return false;
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
    
    private boolean lockSeatInDB(Long showId, Long seatId) {
        UpdateWrapper<Seat> wrapper = new UpdateWrapper<>();
        wrapper.eq("id", seatId).eq("status", 0).set("status", 2);
        return seatService.update(wrapper);
    }
}
```

}

## 八、已实现：乐观锁 + 重试机制

已在 `OrderController.java` 中实现，核心代码如下：

```java
// 重试机制配置
private static final int MAX_RETRY = 3;      // 最大重试次数
private static final long RETRY_DELAY_MS = 50; // 重试间隔(毫秒)

/**
 * 并发安全锁座（乐观锁 + 重试机制）
 */
private boolean lockSeatWithRetry(Long seatId) {
    for (int i = 0; i < MAX_RETRY; i++) {
        // 乐观锁：只有 status=0 才能改成2
        boolean locked = seatService.update(
                new UpdateWrapper<Seat>()
                        .set("status", 2)
                        .eq("id", seatId)
                        .eq("status", 0)
        );
        if (locked) return true;

        // 锁失败，短暂等待后重试
        if (i < MAX_RETRY - 1) {
            try {
                Thread.sleep(RETRY_DELAY_MS);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                return false;
            }
        }
    }
    return false; // 重试次数用尽
}
```

**原理说明：**
```
用户A抢票 → 锁座(status 0→2) → 成功
用户B抢同一座位 → 锁座失败 → 等待50ms → 重试
用户C抢同一座位 → 锁座失败 → 等待50ms → 重试 → 仍失败 → 返回"已被抢购"
```

**为什么能解决并发问题：**
- `UPDATE seat SET status=2 WHERE id=? AND status=0` 是原子操作
- 多个用户同时抢最后一张票，只有一个能成功修改
- 重试机制：抢票失败的用户自动重试3次，提高抢票成功率
